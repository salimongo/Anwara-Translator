# Internationalization Status

## Current Version

The extension now uses Chrome Manifest V3 native localization:

- `E:\skliis\tools\translator-fork\_locales\zh_CN\messages.json`
- `E:\skliis\tools\translator-fork\_locales\en\messages.json`
- `default_locale` remains `zh_CN`, so Chinese browsers keep the existing UI.
- English browsers use the English message catalog automatically.

The popup and standalone reader localize their visible UI at runtime. The content
script reuses the same message lookup without scanning or rewriting host-page
content.

## Deliberately Deferred

The following are intentionally left for later language additions rather than
being mixed into this first pass:

- Japanese, Korean, Spanish, French, German and other locale catalogs.
- Long-form dynamic error/status sentences that include provider error details.
- Project README translations beyond the existing bilingual privacy/license note.
- Locale-specific date and number formatting beyond the browser's existing output.

When adding another language, copy `_locales/en/messages.json`, translate the
values only, validate JSON, then verify the popup, reader, context menu and
selection panel in a browser configured for that locale.
